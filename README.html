📣 کانال بله
مراحل راه‌اندازی
۱
ساخت ربات تلگرام و گرفتن توکن
از طریق BotFather — رایگان و سریع
▾
ℹ️
ربات تلگرام پیام‌های کانال‌های منبع رو دریافت می‌کنه. باید عضو یا ادمین اون کانال‌ها باشه.
  1. در تلگرام، @BotFather رو جستجو کن و باز کن.
  2. دستور /newbot رو بفرست.
  3. یه اسم برای ربات بده (مثلاً: Bridge Bot).
  4. یه یوزرنیم بده که به _bot ختم بشه (مثلاً: mybridge_bot).
  5. BotFather یه پیام با توکن بهت میده که شبیه اینه:
7123456789:AAHxyz-abcdefghijklmnopqrstuvwxyz
⚠️
این توکن رو هیچ‌وقت با کسی به اشتراک نذار. مثل رمز ربانته.
  1. ربات رو به کانال‌های تلگرامی که می‌خوای ادمین کن (یا حداقل عضوش کن تا پیام‌ها رو ببینه).
۲
ساخت ربات بله و گرفتن توکن
از طریق BaleBot — مشابه تلگرام
▾
ℹ️
ربات بله پیام‌ها رو در کانال مقصد منتشر می‌کنه. باید ادمین اون کانال باشه.
  1. در بله، @BaleBot رو جستجو کن.
  2. دستور /newbot رو بفرست و مراحل ساخت رو طی کن.
  3. توکن دریافتی رو کپی کن — فرمتش مثل تلگرامه.
  4. ربات رو در کانال بله‌ات ادمین کن تا بتونه پست بفرسته.
  5. آیدی کانال بله‌ات رو پیدا کن (معمولاً با -100 شروع می‌شه).
💡
برای پیدا کردن آیدی کانال بله، لینک دعوت کانال رو بررسی کن یا از ربات‌های مخصوص ID یاب استفاده کن.
۳
ساخت حساب Cloudflare
رایگانه — نیاز به کارت اعتباری نیست
▾
  1. به cloudflare.com برو و Sign Up کن.
  2. با ایمیل ثبت‌نام کن و آدرستو تأیید کن.
  3. وارد داشبورد بشو — نیازی به دامنه یا پلن پولی نداری.
✓ پلن رایگان کافیه 100K درخواست / روز KV: 1K write / روز
۴
ساخت KV Namespace
فضای ذخیره‌سازی برای تنظیمات و آفست
▾
ℹ️
KV جایی‌ه که Worker توکن‌ها، لیست کانال‌ها و آفست پیام‌ها رو ذخیره می‌کنه.
  1. در داشبورد Cloudflare، از منوی چپ روی Workers & Pages کلیک کن.
  2. از زیرمنو، KV رو انتخاب کن.
  3. روی Create a namespace کلیک کن.
  4. اسمش رو دقیقاً KV بذار (بزرگ).
  5. روی Add کلیک کن.
✅
KV ساخته شد. بعداً باید این Namespace رو به Worker وصل کنی.
۵
ساخت Worker و آپلود کد
قلب پروژه — فایل worker.js رو اینجا آپلود می‌کنی
▾
  1. در داشبورد Cloudflare، روی Workers & Pages کلیک کن.
  2. روی دکمه Create کلیک کن.
  3. گزینه Create Worker رو انتخاب کن.
  4. یه اسم دلخواه بده (مثلاً: tel-to-bale) و روی Deploy کلیک کن.
  5. بعد از ساخت، روی Edit Code کلیک کن.
  6. تمام کد پیش‌فرض رو پاک کن و کد worker.js رو کپی‌پیست کن.
  7. روی Deploy کلیک کن.
⚠️
مطمئن شو که در ابتدای کد export default { وجود داره — این نشون می‌ده Worker رو درست کپی کردی.
۶
وصل کردن KV به Worker
بدون این مرحله Worker کار نمی‌کنه
▾
  1. در صفحه Worker، روی تب Settings کلیک کن.
  2. بخش Bindings رو پیدا کن.
  3. روی Add کلیک کن و KV Namespace رو انتخاب کن.
  4. در فیلد Variable name بنویس: KV (دقیقاً بزرگ)
  5. در فیلد KV Namespace همون Namespace که ساختی رو انتخاب کن.
  6. روی Save کلیک کن.
ℹ️
اگه Variable name اشتباه باشه، Worker پیام KV binding missing میده.
۷
ورود به پنل ادمین و تنظیمات
توکن‌ها و کانال‌ها رو اینجا وارد می‌کنی
▾
  1. آدرس Worker ات رو باز کن:
https://your-worker-name.workers.dev/admin
  1. رمز پیش‌فرض admin1234 رو وارد کن و وارد شو.
  2. در بخش توکن‌ها، توکن ربات تلگرام و بله رو وارد و ذخیره کن.
  3. در بخش کانال مقصد، آیدی کانال بله رو وارد کن (مثلاً -1001234567890).
  4. در بخش کانال‌های منبع، آیدی کانال‌های تلگرامی رو وارد کن — هر کانال یه خط.
  5. روی ذخیره کانال‌ها کلیک کن.
🔐
رمز پیش‌فرض admin1234 رو حتماً در کد عوض کن! خط اول worker.js:
const ADMIN_PASSWORD = "رمز_جدید_خودت";
۸
تست اولیه
ببین همه چیز درسته یا نه
▾
  1. در پنل ادمین، دکمه ▶ اجرا رو بزن.
  2. اگه همه چیز درست باشه، داشبورد آمار رو نشون میده.
  3. یه پیام آزمایشی در کانال تلگرام منبع ارسال کن.
  4. دوباره دکمه اجرا رو بزن — پیام باید در کانال بله ظاهر بشه.
✅
اگه پیام رسید، تبریک! Bridge داره کار می‌کنه. حالا نوبت تنظیم Cron است.
🔍
اگه خطا داری، چک کن: ربات تلگرام در کانال منبع هست؟ ربات بله ادمین کانال مقصده؟ آیدی‌ها درستن؟
۹
تنظیم Cron — اجرای خودکار هر ۵ دقیقه
از cron-job.org استفاده می‌کنیم — رایگانه
▾
⏱️
Cron هر ۵ دقیقه به Worker درخواست میزنه تا پیام‌های جدید رو چک و فوروارد کنه.
  1. به cron-job.org برو و ثبت‌نام کن (رایگانه).
  2. روی Create cronjob کلیک کن.
  3. آدرس Worker ات رو وارد کن:
https://your-worker-name.workers.dev/cron
  1. Schedule رو روی Every 5 minutes بذار.
  2. ذخیره کن و Cron رو فعال کن.
📊
۲۸۸ درخواست در روز — فقط ۰.۳٪ از سقف رایگان Cloudflare. هیچ نگرانی‌ای نیست.
۱۰
گزینه بهتر: Cloudflare Cron Triggers
بدون نیاز به سرویس خارجی — مستقیم از Cloudflare
▾
⭐
Cloudflare Cron Triggers مستقیماً Worker رو صدا می‌زنه بدون HTTP — پایدارتر و بدون مشکل احراز هویت.
  1. در داشبورد Worker، روی تب Settings کلیک کن.
  2. بخش Cron Triggers رو پیدا کن.
  3. روی Add Cron Trigger کلیک کن.
  4. عبارت */5 * * * * رو وارد کن (هر ۵ دقیقه).
  5. روی Add Trigger کلیک کن.
# هر ۵ دقیقه */5 * * * * # هر دقیقه * * * * *
